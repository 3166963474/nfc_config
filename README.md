📄 NFC 配置说明文档（ST25DV + 自定义协议）
=============================

1\. 概述
------

本项目通过 **NFC 标签（ST25DV 系列）** 对设备进行参数配置。  
手机通过 NFC 写入配置数据，设备上电后读取并解析配置，更新运行参数。

👉 手机端只需要：

* 构造配置数据帧（二进制）
* 写入 NFC 存储区

无需参与设备逻辑判断。

* * *

2\. 硬件说明（ST25DV）
----------------

使用芯片：**ST25DV（STMicroelectronics）**

### 主要特点：

* 支持 **NFC Type 5（ISO15693）**
* 支持手机（Android / iOS）读取
* 支持：
  * **RF（NFC）访问**
  * **I2C（MCU访问）**
* 内部带 EEPROM（非易失存储）
* 支持**双接口同时访问（RF + MCU）**

👉 典型流程：

```
手机(NFC) → 写入 EEPROM → MCU(I2C)读取 → 应用配置
```

* * *

3\. 数据存储方式
----------

本项目 **不使用标准 NDEF 文本格式**，而是：

👉 **直接写入原始二进制数据（Raw Binary）**

原因：

* 数据更紧凑
* MCU 解析更简单
* 支持自定义协议（带 CRC）

* * *

4\. 数据帧格式（核心）
-------------

手机需要写入的数据格式如下：

```
[SOF][TIME][TYPE][PAYLOAD][CRC32]
```

* * *

### 4.1 字段定义

| 字段      | 长度     | 说明             |
| ------- | ------ | -------------- |
| SOF     | 1 byte | 固定值 `0xAA`     |
| TIME    | 4 byte | Unix 时间戳（秒，小端） |
| TYPE    | 1 byte | 配置类型           |
| PAYLOAD | N byte | 配置数据           |
| CRC32   | 4 byte | 校验             |

* * *

### 4.2 字节序

👉 **全部采用小端（Little Endian）**

例如：

```
0x12345678 → 78 56 34 12
```

* * *

5\. TYPE 定义
-----------

| 类型     | 值    | 说明   |
| ------ | ---- | ---- |
| MASTER | 0x01 | 主机配置 |
| SLAVE  | 0x02 | 从机配置 |

* * *

6\. PAYLOAD 格式
--------------

* * *

### 6.1 射频参数（所有类型通用）

位于 PAYLOAD 开头：

| 字段   | 类型     | 说明   |
| ---- | ------ | ---- |
| pwr  | uint8  | 发射功率 |
| freq | uint32 | 频率   |
| sf   | uint8  | 扩频因子 |
| bw   | uint8  | 带宽   |
| cr   | uint8  | 编码率  |

👉 总长度：8 字节

* * *

### 6.2 MASTER 配置（TYPE = 0x01）

结构如下：

```
rf (8 bytes)

vehicle_id                 (1)
poll_start_slave           (1)
poll_end_slave             (1)
max_retry                  (1)

reply_timeout_ms           (4)
inter_query_gap_ms         (4)

contend_M                  (1)

cad_idle_tick_ms           (4)
round_sleep_until_tick_ms  (4)
```

👉 总长度：**29 bytes**

* * *

### 6.3 SLAVE 配置（TYPE = 0x02）

```
rf (8 bytes)

vehicle_id   (1)
slave_id     (1)
```

👉 总长度：**10 bytes**

* * *

7\. CRC32 计算规则（非常重要）
--------------------

⚠️ 这里不是标准 CRC32，必须严格按此实现

### 规则：

* 初始值：`0xFFFFFFFF`
* 多项式：`0x04C11DB7`
* 输入单位：**32bit word**
* 数据打包方式：
  * 每 4 字节组成一个 word（小端）
  * 不足 4 字节 → 补 `0xFF`
* 不进行 final xor
* 输出：直接返回 CRC 寄存器值

* * *

### 计算范围：

```
CRC32(SOF + TIME + TYPE + PAYLOAD)
```

👉 不包含 CRC 字段本身

9\. 手机端开发要求
-----------

### 必须支持：

* NFC Type 5（ISO15693）
* 原始数据读写（Raw）

* * *

### Android 建议：

使用：

```
android.nfc.tech.NfcV
```

* * *

### iOS 建议：

使用：

```
CoreNFC (ISO15693)
```

* * *

10\. 写入流程
---------

手机端流程：

```
1. 用户填写配置参数
2. 构造 payload
3. 拼接完整帧
4. 计算 CRC32
5. 写入 NFC EEPROM
```

* * *

11\. 设备端处理逻辑（供理解）
-----------------

MCU 开机：

```
1. 从 EEPROM 读取数据
2. 校验 CRC
3. 比较 TIME（和 Flash）
4. 如果更新 → 写入 Flash
```

* * *

12\. 注意事项
---------

### ⚠️ 1. 必须一次写完整帧

不要分多次写，避免数据不一致

* * *

### ⚠️ 2. CRC 必须正确

否则设备会忽略配置

* * *

### ⚠️ 3. TIME 必须递增

用于判断新旧配置

* * *

### ⚠️ 4. 不要写 NDEF

直接写原始数据

* * *

13\. 推荐测试方式
-----------

* 先用 PC 工具生成 bin
* 手机直接写 bin
* MCU 验证是否识别

* * *

📌 总结
=====

👉 本系统是：

**ST25DV + 自定义二进制协议 + CRC32校验 + 时间戳版本控制**

手机端职责很简单：

👉 **构造帧 + 写入 NFC**
