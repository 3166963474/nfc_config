# 嵌入式通信协议说明文档

1\. 协议概述
--------

本协议用于设备之间的数据通信，采用**定长头 + 变长数据 + 校验**的结构，所有数据均以**字节（Byte）为单位，使用十六进制表示**。

* * *

2\. 数据帧格式
---------

| 字段名称 | 包头   | 长度  | 功能字 | 数据区     | 校验位 |
| ---- | ---- | --- | --- | ------- | --- |
| 字节值  | 0xFC | N   | XX  | N字节     | XX  |
| 下标   | 0    | 1   | 2   | 3 ~ N+2 | N+3 |

* * *

3\. 字段说明
--------

### 3.1 包头（Header）

* **包头：0xFC**
* 用于标识一帧数据的开始

* * *

### 3.2 长度（Length）

* 位于 **下标2**
* 表示**数据段长度**

* * *

### 3.3 功能字（Command）

* 位于 **下标2**
* 用于标识不同的功能或指令类型

* * *

### 3.4 数据区（Payload）

* 位于 **下标3 ~ 校验位前**
* 根据不同功能字定义不同数据结构

* * *

### 3.5 校验位（Checksum）

* 位于**帧最后1字节**
* 用于数据完整性校验

#### ✔ 计算方式：

```
checksum = (包头+长度字+功能字+数据段) % 0xff
```

* * *

4\. 数据存储格式说明
------------

### 小端模式（Little Endian）

当数据由多个字节组成（如 int16、int32）时：

* **低字节在前**
* **高字节在后**

#### 示例：

```
int16 = 0x1234

发送顺序：
0x34 0x12
```

* * *

------------------

### 功能：控制485串口是否打印日志

| 字段  | 包头   | 长度   | 功能字  | 数据段       |            |     | checksum |
| --- | ---- | ---- | ---- | --------- | ---------- | --- | -------- |
| 值   | 0xFC | 0x01 | 0x01 | XX        | 0xXXXXXXXX |     | XX       |
| 说明  |      |      |      | 0：关闭 1：开启 | unix时间戳    |     |          |

详细内容见[功能字表](https://docs.qq.com/sheet/DUUh0dXFiRE16SFJs?tab=BB08J2)，表中表明了已有的指令。带有unix时间戳的参数会保存，重启后仍保持。
